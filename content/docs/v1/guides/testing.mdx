# Testing Scenarios

Payfake's scenario engine lets you simulate any payment outcome deterministically, including multi-step flows like PIN verification, OTP, and 3D Secure.

## Global Scenario Configuration

Configure failure rates and delays for all transactions:

```json
{
  "failure_rate": 0.3,
  "delay_ms": 2000
}
```

This makes 30% of charges fail with a 2 second delay.

## Forcing Specific Outcomes

Force a specific transaction to a terminal state:

```bash
curl -X POST http://localhost:8080/api/v1/control/transactions/REF_xxx/force \
  -H "Authorization: Bearer <jwt_token>" \
  -d '{"status": "failed", "error_code": "CHARGE_INSUFFICIENT_FUNDS"}'
```

## Multi-Step Flow Testing

### Testing PIN Verification Flow

```bash
# 1. Initialize and charge card
curl -X POST http://localhost:8080/api/v1/charge/card \
  -H "Authorization: Bearer sk_test_xxx" \
  -d '{"access_code": "ACC_xxx", "card_number": "4111111111111111", "card_expiry": "12/26", "cvv": "123"}'

# Response returns flow_status: "send_pin"

# 2. Submit PIN
curl -X POST http://localhost:8080/api/v1/charge/submit_pin \
  -H "Authorization: Bearer sk_test_xxx" \
  -d '{"reference": "TXN_xxx", "pin": "1234"}'
```

### Testing OTP Verification Flow

```bash
# 1. Initialize and charge mobile money
curl -X POST http://localhost:8080/api/v1/charge/mobile_money \
  -H "Authorization: Bearer sk_test_xxx" \
  -d '{"access_code": "ACC_xxx", "phone": "+233241234567", "provider": "mtn"}'

# Response returns flow_status: "send_otp"

# 2. Submit OTP
curl -X POST http://localhost:8080/api/v1/charge/submit_otp \
  -H "Authorization: Bearer sk_test_xxx" \
  -d '{"reference": "TXN_xxx", "otp": "123456"}'

# 3. Resend OTP (if needed)
curl -X POST http://localhost:8080/api/v1/charge/resend_otp \
  -H "Authorization: Bearer sk_test_xxx" \
  -d '{"reference": "TXN_xxx"}'
```

### Testing 3D Secure Flow

```bash
# 1. Charge returns flow_status: "open_url" with three_ds_url
curl -X POST http://localhost:8080/api/v1/charge/card \
  -H "Authorization: Bearer sk_test_xxx" \
  -d '{"access_code": "ACC_xxx", "card_number": "4111111111111111", "card_expiry": "12/26", "cvv": "123"}'

# 2. Simulate 3DS completion (normally done by checkout page)
curl -X POST http://localhost:8080/api/v1/public/simulate/3ds/TXN_xxx \
  -d '{"success": true}'
```

## Common Test Scenarios

### Test Successful Payment

```bash
# Reset scenario to defaults
curl -X POST http://localhost:8080/api/v1/control/scenario/reset \
  -H "Authorization: Bearer <jwt_token>"
```

### Test Card Declined

```bash
curl -X POST http://localhost:8080/api/v1/control/transactions/REF_xxx/force \
  -H "Authorization: Bearer <jwt_token>" \
  -d '{"status": "failed", "error_code": "CHARGE_DO_NOT_HONOR"}'
```

### Test MoMo Timeout

```bash
curl -X POST http://localhost:8080/api/v1/control/transactions/REF_xxx/force \
  -H "Authorization: Bearer <jwt_token>" \
  -d '{"status": "failed", "error_code": "CHARGE_MOMO_TIMEOUT"}'
```

### Test Webhook Delivery

Configure a long delay to test webhook handling:

```bash
curl -X PUT http://localhost:8080/api/v1/control/scenario \
  -H "Authorization: Bearer <jwt_token>" \
  -d '{"delay_ms": 10000}'
```

The webhook fires after 10 seconds.

### Test Race Conditions

Force success then immediately verify:

```bash
# Terminal 1: Force success
curl -X POST http://localhost:8080/api/v1/control/transactions/REF_xxx/force \
  -H "Authorization: Bearer <jwt_token>" \
  -d '{"status": "success"}'

# Terminal 2: Simultaneous verify
curl http://localhost:8080/api/v1/transaction/verify/REF_xxx \
  -H "Authorization: Bearer sk_test_xxx"
```

## Resetting After Tests

Always reset the scenario after testing:

```bash
curl -X POST http://localhost:8080/api/v1/control/scenario/reset \
  -H "Authorization: Bearer <jwt_token>"
```
