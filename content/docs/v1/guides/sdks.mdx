# SDKs

Official Payfake SDKs provide idiomatic wrappers around the Payfake API, including full support for multi-step charge flows.

## Authentication Flow

All SDKs follow the same authentication pattern:

1. Create auth client with empty secret key
2. Register or login to get JWT token
3. Call `get_keys()` with token to retrieve actual API keys
4. Create authenticated client with real secret key

<Tabs defaultValue="javascript">
  <TabList>
    <Tab value="javascript">JavaScript</Tab>
    <Tab value="python">Python</Tab>
    <Tab value="go">Go</Tab>
  </TabList>
  <TabPanels>
    <TabPanel value="javascript">
      ```javascript
      // Step 1: Auth client
      const authClient = createClient({ secretKey: '', baseURL });

      // Step 2: Login
      const login = await authClient.auth.login({ email, password });

      // Step 3: Get keys
      const keys = await authClient.auth.getKeys(login.token);

      // Step 4: Real client
      const client = createClient({ secretKey: keys.secretKey, baseURL });
      ```
    </TabPanel>
    <TabPanel value="python">
      ```python
      # Step 1: Auth client
      auth_client = Client(secret_key="", base_url=base_url)

      # Step 2: Login
      login = auth_client.auth.login(email=email, password=password)

      # Step 3: Get keys
      keys = auth_client.auth.get_keys(login.token)

      # Step 4: Real client
      client = Client(secret_key=keys.secret_key, base_url=base_url)
      ```
    </TabPanel>
    <TabPanel value="go">
      ```go
      // Step 1: Auth client
      authClient := payfake.New(payfake.Config{SecretKey: "", BaseURL: baseURL})

      // Step 2: Login
      login, _ := authClient.Auth.Login(ctx, payfake.LoginInput{Email: email, Password: password})

      // Step 3: Get keys
      keys, _ := authClient.Auth.GetKeys(ctx, login.Token)

      // Step 4: Real client
      client := payfake.New(payfake.Config{SecretKey: keys.SecretKey, BaseURL: baseURL})
      ```
    </TabPanel>
  </TabPanels>
</Tabs>

## Handling Multi-Step Charge Flows

Charges may return intermediate `flow_status` values requiring additional input. All SDKs provide methods for each flow step.

## Flow Status Reference

| Status | Method to Call |
|--------|---------------|
| `send_pin` | `submitPIN()` |
| `send_otp` | `submitOTP()` or `resendOTP()` |
| `send_birthday` | `submitBirthday()` |
| `send_address` | `submitAddress()` |
| `open_url` | Open `three_ds_url` in iframe |
| `pay_offline` | Poll `verify()` |
| `success` | Done |
| `failed` | Done |

<Callout type="info" title="Checkout Page Handles This">
  If you're using the hosted checkout page, all multi-step flows are handled automatically. You only need to implement this logic if you're building a custom payment UI.
</Callout>
