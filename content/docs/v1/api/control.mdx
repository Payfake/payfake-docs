# Control API

The Control API is Payfake's power layer — endpoints for manipulating the simulation engine. These have no Paystack equivalent.

All control endpoints require JWT authentication.

**Headers:**
```
Authorization: Bearer <jwt_token>
```

## Scenario Control

<Endpoint method="GET" path="/api/v1/control/scenario" />

### Get Scenario

Returns the current scenario configuration.

**Response:**

```json
{
  "status": "success",
  "message": "Scenario config fetched",
  "data": {
    "id": "SCN_xxxxxxxxxxxx",
    "merchant_id": "MRC_xxxxxxxxxxxx",
    "failure_rate": 0.0,
    "delay_ms": 0,
    "force_status": "",
    "error_code": "",
    "is_active": true
  },
  "code": "SCENARIO_FETCHED"
}
```

<Endpoint method="PUT" path="/api/v1/control/scenario" />

### Update Scenario

Configures the simulation behavior. All fields are optional — only send what you want to change.

**Request Body:**

```json
{
  "failure_rate": 0.3,
  "delay_ms": 2000,
  "force_status": "failed",
  "error_code": "CHARGE_INSUFFICIENT_FUNDS"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `failure_rate` | float | Probability of failure (0.0 - 1.0) |
| `delay_ms` | integer | Artificial delay in milliseconds (max 30000) |
| `force_status` | string | Override all charges to this status |
| `error_code` | string | Error code when status is `failed` |

<Endpoint method="POST" path="/api/v1/control/scenario/reset" />

### Reset Scenario

Resets all simulation settings to defaults (0% failure, 0ms delay).

**Response:**

```json
{
  "status": "success",
  "message": "Scenario config reset to defaults",
  "data": {
    "failure_rate": 0.0,
    "delay_ms": 0,
    "force_status": "",
    "error_code": ""
  },
  "code": "SCENARIO_RESET"
}
```

## Transaction Control

<Endpoint method="GET" path="/api/v1/control/transactions" />

### List All Transactions

Returns all transactions for the merchant with pagination.

**Query Parameters:**

| Param | Default | Description |
|-------|---------|-------------|
| `page` | 1 | Page number |
| `per_page` | 50 | Results per page (max 100) |
| `status` | - | Filter by status |

<Endpoint method="POST" path="/api/v1/control/transactions/:reference/force" />

### Force Transaction Outcome

Forces a pending transaction to a specific terminal state.

**Request Body:**

```json
{
  "status": "failed",
  "error_code": "CHARGE_INSUFFICIENT_FUNDS"
}
```

**Response:**

```json
{
  "status": "success",
  "message": "Transaction outcome forced",
  "data": {
    "id": "TXN_xxxxxxxxxxxx",
    "reference": "your-ref-001",
    "status": "failed"
  },
  "code": "TRANSACTION_FORCED"
}
```

<Callout type="success" title="Deterministic Testing">
  Use the force endpoint for deterministic test scenarios. It bypasses the scenario engine entirely — the outcome is always exactly what you specify.
</Callout>
