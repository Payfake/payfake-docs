# Public Checkout API

Public endpoints are called from the React checkout page in the customer's browser. No authentication required — secured by `access_code`.

<Endpoint method="GET" path="/api/v1/public/transaction/:access_code" />

### Fetch Transaction

Returns transaction details for the checkout page.

**Response (Pending):**

```json
{
  "status": "success",
  "message": "Transaction fetched",
  "data": {
    "access_code": "ACC_xxx",
    "amount": 10000,
    "currency": "GHS",
    "status": "pending",
    "reference": "unique-ref-001",
    "callback_url": "https://yourapp.com/callback",
    "merchant": {
      "business_name": "Acme Store",
      "public_key": "pk_test_xxx"
    },
    "customer": {
      "email": "customer@example.com",
      "first_name": "Kofi",
      "last_name": "Mensah"
    }
  }
}
```

**Response (Already Paid):**

```json
{
  "status": "success",
  "message": "Payment already completed",
  "data": {
    "status": "success",
    "reference": "unique-ref-001"
  }
}
```

<Endpoint method="GET" path="/api/v1/public/transaction/verify/:reference" />

### Verify Transaction

Poll this endpoint to check the status of a transaction. Used for `pay_offline` MoMo flows where the customer approves the payment on their phone.

**Response:**

```json
{
  "status": "success",
  "message": "Transaction verified",
  "data": {
    "reference": "TXN_xxx",
    "status": "success",
    "amount": 10000,
    "currency": "GHS"
  }
}
```

<Endpoint method="POST" path="/api/v1/public/charge/card" />

### Charge Card

Processes a card payment from the checkout page. May return intermediate flow states requiring PIN, OTP, birthday, address, or 3DS.

**Request Body:**

```json
{
  "access_code": "ACC_xxx",
  "card_number": "4111111111111111",
  "card_expiry": "12/26",
  "cvv": "123",
  "email": "customer@example.com"
}
```

<Endpoint method="POST" path="/api/v1/public/charge/mobile_money" />

### Charge Mobile Money

Processes a mobile money payment from the checkout page. May return OTP requirement or `pay_offline` status.

**Request Body:**

```json
{
  "access_code": "ACC_xxx",
  "phone": "+233241234567",
  "provider": "mtn",
  "email": "customer@example.com"
}
```

<Endpoint method="POST" path="/api/v1/public/charge/bank" />

### Charge Bank

Processes a bank transfer payment from the checkout page.

**Request Body:**

```json
{
  "access_code": "ACC_xxx",
  "bank_code": "GCB",
  "account_number": "1234567890",
  "email": "customer@example.com"
}
```

## 3D Secure Simulation

<Endpoint method="POST" path="/api/v1/public/simulate/3ds/:reference" />

### Simulate 3DS

Called after the 3DS simulation page completes. The checkout page opens `/simulate/3ds/:reference` in an iframe, user selects success/fail, and this endpoint processes the result.

**Request Body:**

```json
{
  "success": true
}
```

**Response:**

Returns the final charge outcome with updated `flow_status` (`success` or `failed`).

<Callout type="info" title="3DS Flow">
  When a charge returns `flow_status: "open_url"`, the checkout opens `three_ds_url` in an iframe. After user interaction, a postMessage is sent to the parent, which then calls this endpoint to complete the 3DS verification.
</Callout>
